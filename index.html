Follow this for EVERY demo before showing it to a client. Make sure you've tested it yourself first.

A. Taxi App (Drivemond)

  1. User App: Open → Sign Up with random demo details → Allow location → Try booking a ride
  2. Driver App: Open → Sign Up → Fill demo details → Upload any image → OTP: 000000 (if fails, hit Resend → enter 000000 again) → Add vehicle details
  3. Admin Panel: Login → Go to Vehicle List → Approve the driver's vehicle
  4. Full Test: Keep both apps open side-by-side → Book ride from User App → Accept from Driver App

B. Food Delivery (StackFood)

  1. User App: Open → Sign Up → Allow location → Browse restaurants → Place an order
  2. Restaurant App: Open → Login → Accept/prepare the order
  3. Delivery App: Open → Sign Up → Accept delivery request
  4. Admin Panel: Login → Check orders dashboard, restaurant management

C. Grocery (GroFresh / eGrocer / Hexacom)

  1. User App: Open → Sign Up → Browse products → Add to cart → Place order
  2. Delivery App: Open → Sign Up → Accept delivery
  3. Admin Panel: Login → Check products, orders, delivery management
  4. For eGrocer Multi-Vendor: Also check Seller Panel — show vendor onboarding flow

D. E-Commerce (6Valley)

  1. User App: Open → Sign Up → Browse products → Add to cart → Checkout
  2. Vendor App: Open → Login → Show product listing, order management
  3. Delivery App: Open → Sign Up → Accept delivery
  4. Admin Panel: Login → Show vendor management, product approvals, commission settings

4. Copy-Paste WhatsApp Messages

Pick the right message based on what the lead wants. Click the box to select all text → Copy → Paste in WhatsApp.

Each demo has 3 separate messages — send them one after another in WhatsApp. Message 1: Demo links & credentials → Message 2: Step-by-step testing → Message 3: Cal.com link.

⚠️ DO NOT rapid-fire all 3 messages

Send Message 1 first. Then wait — let the client read it, react, maybe reply. Give it at least 30 seconds to a minute. Then send Message 2. Again, pause. Wait for them to acknowledge or at least give it a moment. Then send Message 3.

The goal is to feel like a real person who's helping them — not a bot dumping links. If you copy-paste all 3 back-to-back in 5 seconds, the client won't read any of them. Let them digest each message before sending the next one.

Best case: they reply after Message 1 or 2, and now you have a conversation going. That's the goal.

5. Sales Flow — Lead to Close

Step 1: Lead Messages (Any Channel)

Send This (Universal Template)

"Hey! Thanks for reaching out to FoundersSuite. Would love to hear about your app idea — let's do a quick chat: https://app.cal.com/founderssuite/30min"

Step 2: On The Call (First 10 Minutes — YOU ASK)

  1. "So tell me — what's the app you're thinking of?"
  2. "Is this for your own business or are you building a marketplace?"
  3. "Who's your target user?"
  4. "How are you solving this problem today without the app?"
  5. "Have you seen any app that's close to what you want?"

Step 3: Show Demo (Next 5 Minutes)

Say This

"Let me show you something — this is similar to what we'd build for you."

→ Use the Decision Matrix (Section 2) to pick the right demo
→ Screen-share the admin panel + walk through the app
→ Narrate: "For your use case, we'd customize X, Y, Z"

Step 4: Close (Last 5 Minutes)

Say This

"Here's what I suggest — we start with an MVP in Phase 1. Your core flow working end-to-end. You launch, validate, and then we add more features in Phase 2."

Phase What's Included Price Timeline
V1 — MVP Core flow: User App + Driver/Vendor App + Basic Admin Panel. One city/region. One vehicle/vendor type. ₹50,000 14-20 days
V2 — Scale Multi-vehicle/vendor, wallet, surge pricing, ratings, push notifications ₹40,000 - ₹60,000 3-4 weeks
V3 — Expand Multi-city/Pan India, multi-language, advanced analytics, referral system ₹50,000 - ₹80,000 4-6 weeks
V4 — Retainer Maintenance, new features, server management, bug fixes ₹30,000 - ₹50,000/mo Ongoing
Pricing Strategy
  • ₹50K is the door opener — never negotiate below this
  • Multi-vendor leads = higher LTV — they always need V2, V3
  • Total lifetime value per client: ₹2-3L+
  • If client asks for discount on V1: "We can do ₹50K for V1, and I'll give you priority scheduling for V2"
  • Never quote all phases upfront — only V1 price on the first call
